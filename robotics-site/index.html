Why this works: Cheapest real option. You own every line of code. Scrappy but honest — judges respect that. If someone has a Pi lying around, cost drops to ~$25 for just the chassis.
Phone-on-Wheels
$15–30
Cheapest possible: RC car + old phone taped on top. Phone runs the camera + CV processing. Laptop sends motor commands via Bluetooth serial to an Arduino on the car.
DifficultyHigh
Demo QualityLow
Time to Working~10+ hours
RiskHigh
SourceAmazon / thrift store
The reality: Cheapest on paper but highest risk. Bluetooth serial is flaky, phone mount is janky, and the RC car motor control is undocumented. Only do this if budget is truly zero.

My recommendation: Go with the iRobot Create 3 if anyone can find one used ($80-120 on eBay). It's the only option where navigation is solved — you focus on the scan logic, not motor wiring. If budget is tight, the RC Chassis + Pi is the honest play — $50, fully owned, shippable in a day. The phone-on-wheels is a last resort only.

Key question for the team: Does anyone already own a Raspberry Pi, an old Roomba, or an RC car? That changes the math. Vote above and let's lock it.

Escort Bot — Follow, Scan, Verify, Log

What you're seeing: A vendor arrives to work on a single node. The bot escorts them to the rack, scans every node top-to-bottom to baseline health, then monitors while they work. It only watches the rack — not the person. If an adjacent node gets touched or something unexpected changes, the bot flags it. When the vendor finishes, it logs what was done and confirms nothing else was disturbed. Privacy-first: no cameras on people, just rack-level awareness.

How the Arm Learns — LeRobot
Show it once. It repeats forever.
A human moves the arm while a webcam records. That data trains a neural network on a cloud GPU. The trained model downloads back to your MacBook and the arm runs autonomously. This is LeRobot — HuggingFace's open-source imitation learning framework. Only used for the SO-ARM101, not the escort bot.

04 / Real Talk
Why this wins.

The Pros

Nobody else is touching hardware. Every other team will demo a dashboard or an LLM wrapper. We're bringing a physical robot to demo day. That alone makes us unforgettable.

It's real DC work. Optic seating, rack inspection, escort logging — these are things we do every single shift. The judges will feel it because it's authentic.

The cost is nothing. ~$350 for a robot arm that can be trained in a weekend. The ROI story writes itself.

It closes the loop. Not a visualization. Not a dashboard. The arm picks up the optic, seats it, and the port goes green. Done.

The Cons

Hardware hasn't shipped yet. 7-14 day window on a 14-day timeline.

Most of us have zero robotics experience.

Training requires hours of teleoperation data nobody's collected yet.

Escort robot platform still TBD.

The pros crush the cons — but only if we execute. No half-speed. We need to ship every piece on time and show up with something no other team can touch. This is how we win it.
