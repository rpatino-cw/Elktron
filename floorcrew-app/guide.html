P

Phase (big text at top)

What the arm is doing right now. Cycles through: Idle (waiting) > Reaching (moving to tray) > Gripping (picking up optic) > Transiting (carrying it) > Inserting (seating into port) > Releasing > Retracting (going home). One full cycle takes ~12 seconds.

J

Joint angles (6 boxes)

Each box is one motor in the arm. The number = rotation angle in degrees. They flash blue when they change. You don't need to understand the exact numbers — just know that movement = the arm is working.

G

Gripper indicator

Shows whether the claw is Open, Closed, or Holding Optic. When it says "Holding Optic" the arm has grabbed a transceiver and is carrying it.

Badge colors:
Ready = idle, waiting
Active = moving
Inserting = critical step
Bottom stats: Torque = motor effort (higher = heavier load). Temp = motor temperature in Celsius (normal: 36-42). Accuracy = placement precision in millimeters. Cycles = total optics seated this session.

P

Phase (big text)

What the bot is doing: Idle (charging) > Dispatched (heading to vendor) > Escorting (walking with them) > Arrived (at rack) > Scanning (checking nodes) > Monitoring (watching for unauthorized touches) > Verified (all clear) > Returning (going back). Full cycle: ~30 seconds in demo.

V

Vendor card

Shows who the bot is escorting: vendor name, ticket number, and which rack + rack unit they're authorized to touch. Appears when a vendor is assigned, disappears when the escort is done.

S

Scan progress bar

Green bar filling up = the bot is scanning each node in the rack from top to bottom. 100% = scan complete. The bar glows while actively scanning.

!

Alert banner (amber/green)

Pops up when the bot detects someone touching an unauthorized rack unit. Amber = "checking..." (contact detected, verifying). Green = "cleared" (false alarm, no state change). A real alert that stays amber = problem.

Bottom stats: Location = where the bot is (aisle name or rack ID). Battery = charge level (starts at 95%, drains slowly). Nodes = how many rack units have been scanned so far.

1

Start Recording

A human demonstrates the task (e.g., seat an optic) while the arm mirrors and records the movement. Each demo = 1 episode. We need ~50 episodes for the arm to learn.

2

Upload Data

Sends all recorded episodes to the cloud for training. Progress bar shows upload status.

3

Train Model

Starts training on a cloud GPU (A100). The model watches all 50 demos and learns the pattern. Takes 2-4 hours, runs unattended. The "Loss" number goes down as it gets better.

4

Deploy

Downloads the trained model and loads it onto the arm. After this, the arm can do the task autonomously — no human needed.

Button behavior: During recording, only the Record button is active (pulsing green). During training, only the Train button is active. Other buttons are dimmed out. This prevents you from accidentally starting two things at once.

Dashboard Modes
What the different connection states mean
D

Demo Mode (what you see now)

No server, no robots. The dashboard generates fake data so you can see how everything looks and moves. Opens from a file — just double-click index.html. All panels animate with realistic simulated cycles.

M

Mock Mode (server, no hardware)

The Python server is running (uvicorn server:app) but no robots are plugged in. Server generates the same simulated data but over a real WebSocket. Good for testing the full stack.

L

Live Hardware Mode

Real robots connected via USB. The arm sends actual joint angles, the escort bot sends real camera frames and sensor readings. This is the goal for demo day.

How to tell which mode you're in: Look at the top bar. "DEMO MODE" (blue) = standalone. "MOCK DATA" (gray) = server without hardware. "LIVE HARDWARE" (green) = the real thing.
